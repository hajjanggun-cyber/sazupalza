# 🎓 사주팔자 프로젝트 심화 학습 로드맵: RAG & 데이터 보강

이 문서는 서비스의 정확도와 풍성함을 높이기 위한 기술적/기획적 공부 방향을 정리한 가이드입니다.

---

## 1. RAG(검색 증강 생성)란?

*   **개념:** AI가 가진 기본 지식 외에, 내가 가진 **전문 문서(블로그 글 등)**를 실시간으로 찾아보고 그 내용을 바탕으로 답변을 생성하는 기술입니다.
*   **비유:** 암기한 내용으로만 답을 쓰는 학생(일반 AI) vs 전공 서적을 옆에 펼쳐놓고 찾아가며 답을 쓰는 학생(RAG).
*   **우리 프로젝트 적용:** 사용자의 사주 결과가 나올 때, `lib/blog/`에 있는 전문 지식을 AI가 읽고 "주인장님의 분석에 따르면..."이라며 훨씬 전문적인 조언을 해줄 수 있습니다.

---

## 2. 학습 경로 A: 직접 RAG 시스템 구현 (개발 중심)

"어떻게 AI가 문서를 찾아내고 말을 하게 만들까?"를 배우는 과정입니다.

1.  **텍스트 임베딩 (Embedding):** 문장을 컴퓨터가 이해하는 숫자(벡터)로 바꾸는 법 배우기.
2.  **벡터 데이터베이스 (Vector DB):** 숫자로 바꾼 문장들을 저장하고 검색하는 창고(Pinecone, Supabase 등) 사용법 익히기.
3.  **오케스트레이션 도구:** AI와 DB를 연결해주는 `LangChain`이나 `LlamaIndex` 라이브러리 공부하기.
4.  **UI/UX:** Next.js에서 실시간 채팅 인터페이스와 AI 답변 연동하기.

---

## 3. 학습 경로 B: AI 활용 데이터 보강 (기획/데이터 중심)

"어떻게 하면 답변의 품질을 전문가 수준으로 끌어올릴까?"를 배우는 과정입니다.

1.  **프롬프트 엔지니어링:** AI에게 사주 전문가의 페르소나를 부여하고, 복잡한 논리 구조를 가진 글을 쓰게 만드는 기법 습득.
2.  **도메인 지식의 알고리즘화:** 
    *   사주 신살(역마살, 도화살 등)의 성립 조건을 데이터화하기.
    *   관상 부위별(눈, 코, 입) 특징에 따른 해석 데이터를 체계적으로 수집하기.
3.  **로컬라이징 (Localization):** 동양 철학 용어를 서구권 사용자도 이해하기 쉽게 문화적으로 번역하고 다듬는 과정.

## 🚀 실습 추천 첫걸음

기술적인 RAG 구현이 어렵게 느껴진다면 아래 순서로 시작해보세요.

1.  **자료 찾기:** 인터넷에서 "사주 신살 보는 법"이나 "관상 눈 모양 해석" 같은 글을 찾습니다.
2.  **AI에게 시키기:** 그 글의 링크나 텍스트를 제미나이에게 주며 **"이 규칙을 우리 사이트 코드에 반영할 수 있게 정리해줘"**라고 요청합니다.
3.  **코드 관찰:** AI가 제안하는 `if`문이나 데이터 구조를 보며 프로그래밍의 기초를 배웁니다.
