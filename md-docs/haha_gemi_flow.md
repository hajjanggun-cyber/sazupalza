# 🌊 사주팔자 프로젝트 작동 흐름 가이드 (Beginner's Guide)

이 문서는 웹 개발 초보자를 위해 본 서비스가 어떻게 시작되고, 사용자의 데이터가 어떻게 처리되어 결과로 나타나는지 그 전체 과정을 쉽게 설명합니다.

---

## 1. 사이트의 대문: index.html은 어디에 있나요?

이 프로젝트는 **Next.js**라는 프레임워크를 사용합니다. 전통적인 방식의 `index.html` 파일은 없으며, 대신 주소(URL)에 따라 **`page.tsx`** 파일이 그 역할을 수행합니다.

*   **대문 페이지 경로:** `app/[locale]/page.tsx`
*   **작동 원리:** 사용자가 사이트에 처음 접속하면 이 파일이 실행되어 메인 홈 화면을 그려줍니다. `[locale]` 부분은 접속한 국가(ko, en 등)에 따라 자동으로 바뀝니다.

---

## 2. 데이터의 여행: 분석 버튼을 누르면 일어나는 일

사용자가 정보를 입력하고 '분석하기'를 누르면 데이터는 다음과 같은 경로를 거칩니다.

### 1단계: 데이터 수집 (Input)
*   **파일:** `components/AnalysisForm.tsx`
*   **내용:** 사용자가 입력한 이름, 생년월일, MBTI 등을 리액트(React)의 바구니(State)에 담습니다.

### 2단계: 주소로 변신 (Encoding)
*   **내용:** 입력된 데이터를 길고 복잡한 문자열로 암호화하여 **고유한 ID**를 생성합니다.
*   **결과:** 웹 주소가 `.../result/eyJnameIjoi...` 처럼 바뀝니다. 이렇게 하면 사용자가 이 주소만 공유해도 다른 사람이 똑같은 결과를 볼 수 있습니다.

### 3단계: 결과 페이지 로드 (Routing)
*   **파일:** `app/[locale]/result/[id]/page.tsx`
*   **내용:** 바뀐 주소(ID)를 보고 Next.js가 결과 페이지를 엽니다. 이 페이지는 주소창에 적힌 ID를 다시 해독(Decoding)하여 원래의 이름과 생일 정보를 알아냅니다.

### 4단계: 두뇌 가동 (Calculation)
*   **경로:** `lib/calculator/`
*   **역할:** 해독된 데이터를 바탕으로 실제 운세를 계산합니다.
    *   `saju-calculator.ts`: 생년월일시로 사주팔자(오행) 계산
    *   `name-calculator.ts`: 이름 획수로 성명학 점수 계산
    *   `result-generator.ts`: 모든 데이터를 합쳐서 최종 영어/한국어 문장 생성

### 5단계: 화면 출력 (Rendering)
*   **내용:** 계산이 끝나면 예쁜 카드 모양 컴포넌트(`SectionCard`, `GuardianCard`)에 데이터를 넣어 사용자에게 보여줍니다.

---

## 4. 블로그 시스템: 글을 쓰면 주소가 어떻게 생기나요?

사이트 하단에 있는 '관련 글'이나 '사주 기초' 같은 블로그 글들은 일반적인 게시판과 조금 다르게 작동합니다.

### 1단계: 원고 작성 (Data)
*   **위치:** `lib/blog/` 폴더 안의 파일들 (예: `saju-posts.ts`, `mbti-posts.ts`)
*   **원리:** 개발자가 이 파일에 글의 제목, 내용, 그리고 **`slug`**라는 주소 이름을 미리 적어둡니다.
    *   예: `slug: 'intro'`, `title: '사주란 무엇인가?'`

### 2단계: 주소 확정 (URL Assignment)
*   **규칙:** `사이트주소/언어/카테고리/주소이름(slug)`
*   **결과:** 위 예시의 경우 주소는 `sajupalza.cc/ko/saju/intro`가 됩니다.

### 3단계: 글 찾아오기 (Fetching)
*   **파일:** `app/[locale]/saju/[slug]/page.tsx`
*   **역할:** 사용자가 `/saju/intro`로 접속하면, 파일 이름인 `[slug]` 자리에 'intro'라는 값이 들어옵니다. 프로그램은 이 'intro'를 가지고 `saju-posts.ts` 데이터에서 해당 글을 찾아 화면에 뿌려줍니다.

---

## 💡 요약
> **입력창(`page.tsx`)** -> **데이터 암호화** -> **주소창 이동** -> **두뇌 엔진(`lib/`) 계산** -> **결과 출력**
>
> **블로그:** **데이터 파일(`lib/blog`)**에 `slug`를 적으면 자동으로 **해당 주소**가 생기고 글이 나옵니다.
