# 📄 사주팔자 프로젝트 디버깅 기록 (haha-gemibingsi)

이 문서는 블로그 현지화 및 데이터 복구 과정에서 발생한 제미나이(AI)의 반복적인 실수와 이를 해결하기 위한 처절한 사투 과정을 기록한 것입니다.

## 🚨 주요 장애 현상
1. **데이터 유실**: 제목 필드 수정 중 3,000자 이상의 풍성한 본문 원고가 요약본으로 덮어씌워짐.
2. **현지화 실패**: 영문 로케일(`/en`)에서도 목록(List) 및 관련 글(Related Posts)에서 한글 제목이 계속 노출됨.
3. **UI 잔재**: 날짜 제거 요청을 받았음에도 불구하고 특정 경로에서 날짜가 계속 표시됨.

## 🛠️ 오류 잡는 과정 (Step-by-Step)

### 1단계: UI 및 날짜 제거 (실패와 재시도)
* **문제**: `BlogPostPage.tsx`에서 날짜 출력 코드를 지웠다고 생각했으나, 조건부 렌더링 로직 오류로 특정 상황에서 여전히 표시됨.
* **해결**: `post.publishedAt`을 출력하는 모든 `<p>` 태그와 관련 로직을 물리적으로 완전 삭제.

### 2단계: MBTI 데이터 복구 (본문 누락 사건)
* **문제**: 제목(`title`) 필드를 영어로 바꾸는 과정에서 `write_file` 도구로 파일 전체를 덮어쓰며 기존의 풍성한 국/영문 본문 데이터를 날려버림.
* **해결**: 기획안 원고를 대조하여 23번~27번 포스트의 내용을 다시 처음부터 3,000자 이상으로 정밀하게 재작성 및 복구.

### 3단계: 영문 모드 한글 제목 노출 (데이터 구조의 맹점)
* **문제**: `BlogPostPage.tsx`의 로직은 `seoTitleEn`을 우선하도록 짰으나, `seoTitleEn`이 비어있거나 `title` 필드가 한글로 하드코딩되어 있어 영문 모드에서도 한글이 튀어나옴.
* **해결**: 
    - `lib/blog/mbti-posts.ts`의 `title` 필드 자체를 영문으로 강제 교체.
    - `app/[locale]/mbti/page.tsx` 목록 페이지의 렌더링 로직을 `(post.seoTitleEn || post.title)`로 강화하여 한글 노출 가능성 차단.

### 4단계: 관련 글(Related Posts) 한글 노출 (이중 구멍)
* **문제**: 개별 포스트 데이터 하단에 정의된 `relatedPosts` 배열 내의 제목들이 한글로 하드코딩되어 있었음.
* **해결**: `saju-posts.ts`, `mbti-posts.ts`, `bokhap-posts.ts` 등 모든 데이터 파일의 `relatedPosts` 타이틀을 일일이 영어로 수동 수정.

## 💡 최종 교훈
* **AI의 자만 방지**: 코드를 고쳤다고 믿지 말고, 실제 파일의 끝까지 다시 읽어(`read_file`) 데이터 유실 여부를 반드시 확인해야 함.
* **로컬라이징의 철저함**: UI 로직만 고친다고 해결되지 않음. 데이터 레이어(`title`, `relatedPosts`)에 박힌 하드코딩된 한글을 모두 제거해야 완벽한 글로벌 서비스가 가능함.
