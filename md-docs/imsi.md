## 1. 최종 판단

오프스 분석은 큰 방향에서 맞다. 특히 "AI 패턴 반복 문제의 핵심 규칙은 이미 post-codex-command.md에 있다"는 판단은 사실에 가깝다. 현재 문서에는 도입부 변주, H2 순서 반복 금지, 문단 길이 변주, FAQ 복사 금지, CTA/결론 문구 분산, 시리즈 제목 반복 금지가 이미 들어 있다.

다만 "추가해야 할 규칙은 3개뿐"이라는 결론은 조금 좁다. 실제로는 오프스가 지적한 3개 외에도 아래 보강이 필요하다.

1. 이미 있는 규칙을 실제 작성 단계에서 강제하는 검수 게이트
2. `Series H2 Template Rules`가 앞쪽의 반복 금지 규칙을 약화시키지 않도록 하는 충돌 해소 문구
3. 오래된 글과 신규 글을 구분해 정비하는 적용 범위 규칙
4. `Start Here`에서 시리즈 글 작성 시 반드시 14~15번 섹션을 보게 하는 흐름 보강

즉, 문서 자체는 방향이 좋지만 "규칙이 많고 중복되어 있어서 실제 작성자가 놓치기 쉬운 구조"다. 단순히 규칙을 더 많이 추가하는 것보다, 하드 체크리스트와 금지 패턴을 위쪽에 압축해 넣는 수정이 더 효과적이다.

## 2. 오프스 분석 중 맞는 부분

### 2-1. 반복 방지 핵심 규칙은 이미 존재한다

오프스가 지적한 항목 대부분은 실제로 문서에 있다.

| 오프스 지적 | 현재 문서 근거 | 판단 |
|---|---:|---|
| CTA/결론 반복 금지 | 0-AA-4, line 88 | 맞음 |
| H2 순서 동일 금지 | 0-AA-5, line 92 | 맞음 |
| 문단 길이 변주 | A2, line 200 | 맞음 |
| FAQ 복사 금지 | A2, line 209 / 5-X, line 571 | 맞음 |
| 시리즈 제목 복사 금지 | 5-1, line 517 / 14-2, line 788 | 맞음 |
| 도입부 변주 | 0-AA-4, line 84 / 3-X, line 448~456 | 맞음 |
| 첫 문단 공허한 도입 금지 | 3-X, line 454 | 맞음 |

따라서 "반복형 AI 글을 막는 원칙이 문서에 없다"는 문제는 아니다. 문제는 "있는 원칙이 실제 산출물에서 통과 조건으로 작동하지 않는다"는 쪽이다.

### 2-2. description의 분량 표현 금지 규칙은 실제로 빠져 있다

현재 `description` 규칙은 line 530~535에 있다. 내용은 "실제 다루는 범위 요약", "제목과 다른 내용 약속 금지", "키워드 목록처럼 나열 금지" 정도다. 하지만 `3,000자`, `2,500자`, `전문 해설`, `전문 데이터` 같은 분량/품질 과시형 문구를 금지하지 않는다.

실제 글 데이터에서도 문제가 확인된다.

- `3,000자` 검색 결과: 전체 `lib/blog/*.ts`에서 54회
- `description`과 첫 문단에 "3,000자 전문 해설", "3,000자 전문 데이터"가 반복됨

이건 반드시 문서에 명시적으로 막아야 한다. 특히 line 185와 line 428의 내부 분량 기준은 "작업 내부 품질 하한선"이어야지, 독자에게 노출되는 description이나 본문 첫 문단에 들어가면 안 된다.

### 2-3. seoTitle 후위어 반복 제한도 실제로 빠져 있다

현재 문서 line 517에는 "시리즈 글이라도 일간명/일주명/주제명만 바꾼 반복 제목을 피한다"가 있고, line 518에는 키워드 스터핑 제목 금지가 있다. 하지만 사이트 전체에서 `완벽 가이드`, `총정리`, `완전정복`, `완전 해설` 같은 후위어가 과도하게 반복되는 것을 막는 수치형 또는 배치형 규칙은 없다.

실제 글 데이터에서도 반복이 확인된다.

- `완벽 가이드`: 18회
- `완전정복`: 42회
- `총정리`: 18회

따라서 오프스가 말한 "후위 표현 분산 규칙 누락"은 맞다.

### 2-4. CTA/관련글 구조 반복 문제도 실제로 있다

현재 문서에는 CTA 문구 반복 금지(line 88, line 92)와 CTA성 링크 앵커 규칙(line 618)이 있다. 하지만 "HTML 블록의 위치, 역할, 종류, 문구를 어떻게 변주할지"는 충분히 구체적이지 않다.

실제 글 데이터:

- `linear-gradient`: 전체 `lib/blog/*.ts`에서 167회
- `related-posts-inline`: 전체 `lib/blog/*.ts`에서 171회
- 최근 추가된 30~36번 글에서는 구식 inline gradient는 줄었지만, `cta-box` 안의 CTA 문구가 `사주 분석 시작하기` / `Start a Saju Analysis`로 거의 반복됨

단, 여기서 "픽셀 단위로 디자인을 매번 다르게 해야 한다"는 식으로 가면 안 된다. 디자인 시스템 관점에서는 같은 컴포넌트가 같은 스타일을 갖는 것이 정상이다. 진짜 문제는 CSS가 같은 것이 아니라, 본문 안에 inline style이 대량으로 박혀 있고 CTA의 역할/위치/문구가 기계적으로 반복되는 것이다.

## 3. 오프스 분석 중 보정이 필요한 부분

### 3-1. "실제 글이 전혀 따르지 않는다"는 표현은 과하다

오래된 글에는 오프스 지적이 강하게 맞다. `3,000자`, inline gradient CTA, `완벽 가이드`, `완전정복`, "Conclusion" 패턴이 실제로 많이 남아 있다.

하지만 최근 추가된 33~36번 글은 예전 글보다는 낫다.

- `saju-unknown-birth-time`, `daewoon-change`, `day-branch-reading`, `ilgan-career-table`은 표, 체크리스트, 한계 설명, 주제 분리를 꽤 반영하고 있다.
- H2도 30~32번 일주 글처럼 완전히 동일한 5개 템플릿만 반복하지는 않는다.

그래도 남은 문제는 있다.

- CTA 문구가 거의 동일하다.
- 결론 문단이 `결론적으로` / `In short` 패턴으로 반복된다.
- 중간 CTA 위치가 거의 고정되어 있다.
- 일부 EN 도입부는 `This guide...`, `This article...` 패턴이 반복된다.

정확한 표현은 "전체 누적 글에는 규칙 위반이 많고, 최근 글도 일부 반복 패턴이 남아 있다"가 맞다.

### 3-2. "HTML 메타 구조를 픽셀 단위로 변형"은 권장하지 않는다

오프스가 말한 "관련글 박스가 픽셀 단위로 동일"은 문제의 핵심이 아니다. 일관된 UI 컴포넌트는 오히려 유지보수와 UX 측면에서 맞다.

수정 방향은 아래가 더 정확하다.

- 같은 inline style을 글마다 박는 방식은 줄인다.
- 가능하면 `related-posts-inline`, `cta-box` 같은 공통 class나 렌더러 컴포넌트로 통일한다.
- 다만 글마다 CTA의 목적, 문구, 위치, 앞뒤 문맥은 바꾼다.
- 관련글 박스도 "함께 보면 좋은 글"만 반복하지 말고 "먼저 읽을 글", "다음 단계", "비교해서 볼 글", "이론 배경" 등 역할을 나눈다.

즉, CSS 랜덤화가 아니라 "콘텐츠 구조의 기능적 변주"가 필요하다.

## 4. post-codex-command.md 자체에서 수정 권장되는 부분

### 4-1. 누락 규칙 1: seoTitle 후위어 사용 제한

추가 권장 위치:

- `## 5. SEO Rules`
- `### 5-1. 제목` 바로 아래

추가해야 할 취지:

- `완벽 가이드`, `총정리`, `완전정복`, `완전 해설`, `A to Z`, `complete guide`, `ultimate guide` 같은 후위어는 같은 카테고리나 연속 발행 배치에서 반복 사용하지 않는다.
- 제목 후위어는 검색 의도가 실제로 "정리/가이드/표/보는법/차이"인지에 맞춰 선택한다.
- 같은 배치 5개 글 중 1개 정도만 강한 후위어를 허용하고, 나머지는 구체 정보형 제목으로 분산한다.
- 시리즈 글은 `주제명 + 후위어` 복사가 아니라, 각 글의 차별 포인트를 제목에 넣는다.

### 4-2. 누락 규칙 2: description과 첫 문단에 분량 표현 금지

추가 권장 위치:

- `### 5-4. 메타 설명`
- `## 3. Writing Format Rules`
- `## 0-AC. AdSense Low-Value Content Guardrails`

추가해야 할 취지:

- `3,000자`, `2,500자`, `1,000 words`, `전문 데이터`, `전문 해설`, `완벽하게`, `상세히 공개` 같은 분량/품질 과시형 표현을 `description`, 첫 문단, CTA에 쓰지 않는다.
- 내부 분량 기준은 작성자 검수용이며 독자 노출 문구로 전환하지 않는다.
- description은 분량이 아니라 "무엇을 판단할 수 있는지", "어떤 범위를 다루는지", "어떤 한계를 인정하는지"를 요약한다.

### 4-3. 누락 규칙 3: CTA/관련글 블록의 기능적 변주

추가 권장 위치:

- `## 6. Internal Linking Rules`
- 또는 `## 3. Writing Format Rules` 아래에 `CTA and Related Block Variation Rules` 신설

추가해야 할 취지:

- 같은 카테고리 연속 글에서 CTA 문구를 그대로 반복하지 않는다.
- CTA는 "분석 시작", "내 차트 확인", "오행 균형 확인", "관계 패턴 확인", "직업 흐름 점검"처럼 글의 검색 의도에 맞춰 다르게 쓴다.
- 같은 위치에만 CTA를 넣지 말고, 글 성격에 따라 원리 설명 뒤 / 체크리스트 뒤 / 한계 설명 뒤 중 하나를 선택한다.
- 관련글 박스 제목도 `함께 보면 좋은 글`만 반복하지 않고, `먼저 읽을 이론`, `다음 단계`, `비교해서 볼 글`, `실전 적용 글`처럼 역할을 분리한다.
- 단, 스타일은 공통 class로 유지해도 된다. CSS를 글마다 무작위로 바꾸는 것은 권장하지 않는다.

### 4-4. 추가 권장 규칙: 작성 완료 전 패턴 감사 게이트

오프스 분석에서 가장 빠진 핵심은 "강제 장치"다. 이미 있는 규칙이 실제 글에서 무시되는 이유는 마지막 검수 항목이 충분히 구체적이지 않기 때문이다.

추가 권장 위치:

- `## 8. Completion Rules`
- `### [A4] 완료 전 추가 품질 체크`

추가해야 할 체크:

- 같은 카테고리 최근 5개 글의 `seoTitle` 후위어가 반복되지 않는가?
- 같은 카테고리 최근 5개 글의 첫 문단 시작 방식이 반복되지 않는가?
- 같은 카테고리 최근 5개 글의 CTA 문구가 동일하지 않은가?
- `description`에 `3,000자`, `2,500자`, `words`, `전문 데이터`, `완벽` 같은 분량/과시 표현이 없는가?
- 새 글 H2가 같은 시리즈 템플릿을 그대로 복사하지 않았는가?
- FAQ 중 최소 1개는 본문 요약이 아니라 오해/예외/한계를 다루는가?

이 검수 게이트는 문서 추가보다 중요하다. 지금 문서는 원칙은 많지만, 마지막에 "이걸 실제로 검사하라"는 압력이 약하다.

### 4-5. 문서 내부 충돌: 표준 H2 템플릿이 반복 금지를 약화시킨다

문서 line 816 이후 `Series H2 Template Rules`는 유용하지만 위험하다. line 817에는 "템플릿을 기준으로 작성하되 변주한다"고 되어 있으나, 실제 작성자는 아래 5개 H2를 그대로 복사하기 쉽다.

특히 일주 시리즈는 line 835~839의 구조가 최근 30~32번 글에 거의 그대로 반영되어 있다.

수정 방향:

- `표준 H2 구조`를 "복사 가능한 제목"이 아니라 "필수로 다룰 해석 단위"라고 명시한다.
- H2 제목 자체는 매 글마다 고유하게 바꾸도록 강제한다.
- 같은 시리즈에서 H2 순서도 필요하면 바꿀 수 있게 한다.
- 예: 일주 글이라도 어떤 글은 2026 이슈를 앞에 두고, 어떤 글은 연애 패턴을 먼저 두고, 어떤 글은 직업·재물 리스크를 중심으로 잡을 수 있어야 한다.

### 4-6. `Start Here`에 시리즈/신뢰성 규칙이 빠져 있다

현재 `Start Here`는 line 253~268에 있고, 0-AA, 0-AB, 0-A, 0-AC, 구조, 필수 필드, SEO, 내부 링크, 완료, 빌드를 보게 되어 있다.

하지만 시리즈 글 작성 시 중요한 섹션인 14, 15, 18이 기본 순서에 없다.

수정 방향:

- 시리즈 글이면 `## 14. Series Cannibalization Prevention Rules`와 `## 15. Series H2 Template Rules`를 반드시 확인하도록 추가한다.
- 운세/건강/재물/결혼/이직 등 민감 주제면 `## 18. E-E-A-T Signal Rules`도 확인하도록 추가한다.

이건 실제 작업 누락을 줄이는 데 효과가 크다.

### 4-7. 오래된 글 정비 범위 규칙이 필요하다

문서가 앞으로의 글 작성 규칙만 다루면, 이미 쌓인 오래된 문제는 계속 남는다.

실제 현재 데이터에는 아래 누적 문제가 있다.

- `3,000자`: 54회
- `linear-gradient`: 167회
- `related-posts-inline`: 171회
- `완벽 가이드`: 18회
- `완전정복`: 42회
- `총정리`: 18회

따라서 문서에 "신규 글은 즉시 적용, 기존 글은 우선순위 큐로 정비"라는 운영 규칙을 추가하는 편이 좋다.

추천 정비 우선순위:

1. description에 `3,000자`가 들어간 글
2. title/seoTitle에 `완벽 가이드`, `완전정복`, `총정리`가 많은 핵심 카테고리 글
3. inline gradient CTA가 남아 있는 오래된 글
4. `Conclusion` H2를 반복하는 EN 글
5. Search Console 노출은 있는데 CTR 낮은 글

## 5. 실제 글 기준으로 본 문제 우선순위

### 최우선: description의 분량/과시 문구 제거

이건 문서 수정과 실제 글 수정 모두 필요하다. `description`은 검색 결과 스니펫 후보가 될 수 있으므로, "3,000자 전문 해설" 같은 문구는 품질 신호보다 광고 문구처럼 보일 가능성이 크다.

우선순위가 높은 이유:

- 검색자에게 실제 도움보다 분량을 팔고 있다.
- Google 공식 기준의 "내용 요약" 목적과 다르다.
- 사이트 전체에 같은 문구가 반복돼 복제 느낌을 만든다.

### 2순위: 구식 inline CTA/relatedPosts 스타일 정리

오래된 글에는 inline style로 박힌 CTA가 많다. 이건 콘텐츠 품질 문제이면서 유지보수 문제다.

다만 해결 방향은 "글마다 디자인 다르게"가 아니다.

- 공통 class 또는 렌더러 구조로 정리
- CTA 문구와 위치만 글별로 변주
- 관련글 박스의 역할 라벨을 글별로 변주

### 3순위: 시리즈 H2 템플릿의 복사 사용 차단

최근 30~32번 일주 글은 line 835~839 템플릿을 거의 그대로 따랐다. 문서에는 변주하라고 되어 있지만, 실제 출력은 템플릿 복사에 가깝다.

해결 방향:

- 템플릿은 "포함할 정보 단위"로 바꾼다.
- H2 제목은 매 글 고유 문장으로 재작성하게 한다.
- 같은 시리즈 연속 작성 시 직전 3개 글의 H2와 비교하게 한다.

### 4순위: 결론 문장 패턴 정리

최근 글에도 `결론적으로`, `In short`가 반복된다. 이건 치명적이지는 않지만 AI 패턴 느낌을 만든다.

문서에 "결론/마무리 문장은 반드시 있어야 하는 것이 아니며, 마지막 H2 안에서 자연스럽게 닫아도 된다"는 규칙을 추가하면 좋다.

## 6. 문서 수정 방식 제안

규칙을 길게 더 붙이면 또 안 읽힐 가능성이 높다. 아래 방식이 낫다.

### 방식 A: 위쪽에 "Hard Pattern Ban" 신설

`0-AA` 바로 아래 또는 `0-AC` 앞에 20줄 정도의 강한 금지 목록을 추가한다.

핵심 금지:

- description에 분량 표현 금지
- seoTitle 후위어 반복 금지
- 같은 CTA 문구 연속 사용 금지
- 같은 관련글 박스 라벨 반복 금지
- 시리즈 H2 템플릿 제목 복사 금지
- `결론적으로`, `In short`, `This guide explains` 같은 마무리/도입 패턴 반복 금지

### 방식 B: 기존 섹션에 분산 추가

이 방식은 문서 구조상 자연스럽지만, 실제 작성자가 놓칠 가능성이 있다.

- 5-1 제목: 후위어 제한
- 5-4 메타 설명: 분량 표현 금지
- 6 내부 링크: CTA/관련글 문구 변주
- 8 완료 규칙: 패턴 감사 게이트
- 15 시리즈 템플릿: 복사 금지 강화

추천은 A+B 혼합이다. 위쪽에는 짧은 금지 목록을 두고, 아래 기존 섹션에는 세부 규칙을 넣는 것이 좋다.

## 7. 오프스 분석에 대한 최종 평가

오프스 분석은 방향성이 좋다. 특히 "문서 자체보다 실제 글 수정이 더 시급하다"는 결론은 맞다.

하지만 내가 보기에는 이렇게 정리하는 것이 더 정확하다.

1. `post-codex-command.md`에는 반복 방지 원칙이 이미 충분히 많다.
2. 그래도 오프스가 말한 3개 누락 규칙은 실제로 추가할 가치가 있다.
3. 추가로, 문서에는 "강제 검수 게이트"와 "시리즈 템플릿 충돌 해소"가 더 필요하다.
4. 실제 글은 오래된 누적 글에서 문제가 훨씬 크고, 최근 33~36번도 CTA/결론 패턴은 아직 반복된다.
5. CSS를 글마다 다르게 만들 필요는 없고, 디자인은 공통화하되 문구·위치·역할을 변주해야 한다.
6. 문서 수정 후에는 기존 글 정비 작업을 별도 배치로 진행해야 한다. 그렇지 않으면 새 규칙만 좋아지고 사이트 전체 품질은 그대로 남는다.

## 8. 권장 결론

`post-codex-command.md`는 수정할 필요가 있다. 다만 대규모로 갈아엎을 필요는 없다.

수정 우선순위는 다음이다.

1. `description` 분량 표현 금지 규칙 추가
2. `seoTitle` 후위어 반복 제한 추가
3. CTA/관련글 블록의 문구·위치·역할 변주 규칙 추가
4. 완료 전 패턴 감사 체크리스트 추가
5. `Series H2 Template Rules`를 "복사용 제목"이 아니라 "포함할 정보 단위"로 재정의
6. `Start Here`에 시리즈 글 작성 시 14~15번, 민감 주제 작성 시 18번을 보라는 조건 추가
7. 기존 글 정비 우선순위 규칙 추가

이 정도면 오프스가 지적한 핵심 문제를 문서 차원에서 막을 수 있다. 하지만 문서 수정만으로 끝내면 안 되고, 이후 실제 글 데이터에서 `3,000자`, `완벽 가이드`, inline gradient CTA, 반복 결론 H2를 정리하는 작업이 필요하다.
